# 댓글 운영 가이드

## 기본 철학

이 채널에서 좋은 반론은 환영받는 참여다. 반론은 상대를 이기기 위한 말이 아니라 질문을 더 선명하게 만드는 말이다.

공개 문구에서는 `겐세이` 대신 `좋은 반론`이라는 표현을 쓴다. 문제의식은 살리되, 처음 들어오는 사람이 거칠게 느끼지 않도록 한다.

## 좋은 댓글 기준

좋은 댓글은 아래 셋 중 하나를 포함한다.

- 이유: 왜 그렇게 생각하는지 설명한다.
- 경험: 자기 삶에서 본 장면을 붙인다.
- 책 내용: 책의 문장, 장면, 개념과 연결한다.

예시:

```text
저는 조금 다르게 봅니다. 쾌락 자체가 문제라기보다 선택권을 잃는 순간이 문제라고 생각합니다. 그래서 도파민을 완전히 끊기보다 거리 조절이 핵심인 것 같습니다.
```

## 아쉬운 댓글 기준

아쉬운 댓글은 바로 삭제하지 않고, 가능하면 질문으로 다시 열어준다.

예시 댓글:

```text
그건 아닌 듯
```

운영자 답글:

```text
어느 부분이 다르게 느껴졌나요? 이유나 경험을 하나만 붙여주면 토론이 더 재밌어질 것 같습니다.
```

## 숨김/삭제 기준

아래 댓글은 숨김 또는 삭제한다.

- 인신공격
- 조롱
- 혐오 표현
- 낙인찍기
- 반복 도배
- 토론과 무관한 홍보

기준 문장:

```text
사람을 공격하는 댓글은 숨깁니다. 생각에 대한 반론은 환영합니다.
```

## 운영자 답글 템플릿

### 좋은 반론을 받았을 때

```text
좋은 반론입니다. 이 관점에서 보면 질문이 조금 바뀌네요.
```

### 추가 근거를 요청할 때

```text
이 의견이 더 궁금합니다. 이유나 책의 장면 하나를 붙이면 더 깊게 이어질 것 같아요.
```

### 댓글끼리 충돌할 때

```text
두 의견 모두 질문을 다르게 보고 있는 것 같습니다. 여기서는 사람보다 생각을 기준으로 이어가면 좋겠습니다.
```

### 토론을 마무리할 때

```text
오늘 댓글을 보면 핵심은 A와 B 사이의 균형으로 모이는 것 같습니다. 다음 게시물에서 이 지점을 다시 다뤄보겠습니다.
```

## 인스타그램 기능 활용

인스타그램의 숨김 단어, 댓글 숨김, 댓글 켜기/끄기 기능을 운영 보조 장치로 사용한다. 기본 방향은 댓글을 막는 것이 아니라, 토론을 무너뜨리는 표현만 줄이는 것이다.

숨김 단어에는 계정 운영 중 반복적으로 문제를 만드는 조롱 표현과 혐오 표현을 추가한다. 단, 단순한 반대 의견이나 강한 문제 제기는 숨김 단어로 막지 않는다.
